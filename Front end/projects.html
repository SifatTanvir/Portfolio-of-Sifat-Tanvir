LLM Compression

Why LLM Compression is necessary

Due to the model complexity, exceptional hardware requirements, large training data and lengthy inference time, large language models (LLMs) are not scalable.

Scaling of LLMs involves several challenges such as trade-off between the expenses, ensuring compliance with data regulations and the performance, increased model complexity and latency.

Successful scaling of LLMs can lead to the proper balancing of the expenses, model complexity, latency and performance.

This shows the comparative analysis among several state-of-the-art LLM compression techniques such as Parameter Pruning, Dynamic Quantization, and Mixed Precision Quantization on the BERT (bert-base-uncased) model and analyzes which technique yields the best performance tackling the scalability issues and reveals their limitations.

The research also introduces a hybrid approach of Pruning and Dynamic Quantization, and shows how the proposed approach performs compared to the state-of-the-art approaches. The implementation of the hybrid approach is also highlighted in this work.

How Quantization Aware Training (QAT) affects the models is discussed at the end.

Falling Object Catcher

Arcade-style Android game: catch good items, avoid bad ones, with a starfield backdrop and touch-and-drag controls — built with Jetpack Compose and Material 3.

Download APK

🛠️ Tech stack

  • Kotlin — primary language
  • Jetpack Compose — modern declarative UI toolkit
  • Material3 — Material Design 3 components
  • ViewModel — state management with lifecycle awareness
  • Coroutines & Flow — asynchronous game loop

📱 Features

  • Smooth 60 FPS gameplay
  • Touch and drag controls
  • Colorful arcade-style visuals
  • Animated starfield background
  • Visual feedback for good/bad items
  • Game over screen with restart option

📋 Requirements

  • Android Studio Arctic Fox (2020.3.1) or newer
  • Kotlin 1.9.20
  • Android SDK 24+ (target 34)
  • JDK 17

📖 Game rules

Core loop. Objects fall; the basket moves horizontally. A catch registers when an object is horizontally over the basket and vertically in the catch band (about 82–95% of screen height).

Your score counts good catches (cash 💵 is scored like any other good item, not as a separate “dollar” tally).

Scoring (non–Color Match). Base +1 per good catch. With an active Perfect combo, each good catch adds points equal to the current consecutive count (e.g. 7 in a row on the same type → +7 that catch).

Lives. You start with 3. Catching a bad item costs one life; at zero lives it is game over. Immunity (🛡️): when active, the next bad hit uses up the shield instead of a life (positive catch sound). If you take a bad hit with no shield, you lose a life, your consecutive good streak resets to zero, and Perfect and Variety combos reset.

Mode-specific notes
  • Color Match: Basket color vs object (incl. multi-color rules). Match → score like success; mismatch → −1 life. No buff/nerf spawns; Perfect/Variety disabled. Basket color rotates every 5 s among 🔴🟡🟢⚫⚪. Multi-color: 🔥 matches red or yellow; 🔪 black or white; 🚀 red or white (Color Match only).
  • Survival: Difficulty rises every 60 s (faster falls, higher bad spawn capped at 60%). Spawn rate increases with level (delay floor ~400 ms with jitter). 15% of “bad lane” spawns can be 🛡️ instead of a true bad object.
  • Classic / Time Attack / Mystery: ~75% good spawn; of the remainder, 20% chance of 🛡️ instead of a true bad item.
  • Mystery: Objects show “?” until they are far enough down the screen (about 65% of the way), then reveal. The lives row never shows more than three hearts.
Combos (disabled in Color Match)
  • Perfect (same good type in a row): The streak grows when you repeat the same good emoji; switching to another good resets the streak to 1 on the new type. After five or more same-type good catches in a row, the combo is “active”; while active, each good catch adds points equal to the current streak length. A bad hit that costs a life resets Perfect (and Variety). The UI can show “NX” when active.
  • Variety: The game tracks which of the six good types you have caught at least once. Catch all six before a bad hit clears the list. Re-catching a type you already have still counts toward completion. Finishing the full set awards +50 points, bumps the variety completion count, shows a short bonus, then clears so you can run it again. The label shows completions as “V : N”.
  • Stats: Max perfect streak stored for leaderboard/stats persistence.
Other systems
  • Leaderboard (top 3) on game over by score.
  • Statistics / DB: emoji counts, variety totals, max perfect, play time, good spawned/caught, bad caught, top scores per mode.
  • Save slots: load/save; finishing a run from a slot deletes that save.
  • Time Attack: 120 s timer (time at zero ends the run). You have a very large life count, but each bad catch still reduces lives in the background, so it is effectively a cushion, not invincibility. Spawns speed up as time runs out. Same object mix as Classic and Mystery (not the Color Match or Survival tables).
